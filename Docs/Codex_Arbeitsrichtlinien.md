# Codex Arbeitsrichtlinien

Dieses Dokument hält verbindliche Arbeitsregeln fuer die weitere Zusammenarbeit mit Codex in diesem Projekt fest.

## Zweck

Die Richtlinien dienen dazu, wiederkehrende Erwartungen klar zu dokumentieren, damit Umsetzung, Kommunikation und Uebergabe konsistent bleiben.

## Verbindliche Regeln

1. Nach jeder inhaltlichen Umsetzung auf einen Prompt soll Codex eine passende Commit-Message auf Englisch vorschlagen.
2. Commit-Messages sollen kurz, praezise und in einem professionellen Stil formuliert sein.
3. Wenn eine Aenderung mehrere Bereiche betrifft, soll die Commit-Message den wichtigsten fachlichen Schwerpunkt priorisieren.

## Format fuer Commit-Message-Vorschlaege

Bevorzugtes Format:

```text
type: short description
```

Beispiele:

```text
feat: redesign solution section layout
fix: make pain point cards backward compatible
style: update landing page color system
```

## Erweiterung

Weitere projektbezogene Regeln koennen in diesem Dokument fortlaufend ergaenzt werden.
